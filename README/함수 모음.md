# 문자열 확인(조회) 함수


```java
 length() : 길이 보기
String str = "hello";
System.out.println(str.length());

 결과: 5
```
```java
 charaAt() : 특정 위치 문자 보기 **
String str = "hello";
System.out.println(str.charAt(1));

결과 e      (index 0부터 시작해서 hello중 "e" 나옴)
```
```java
 contains() : 포함 여부 확인
String str = "hello";
System.out.println(str.contains("ell"));

결과 : true
```
```java
indexOf() : 위치 찾기 **
String str = "hello";
System.out.println(str.indexOf("l"));

결과 2      (index 0부터 시작해서 "hello" l은 2번째 위치)
```
```java
 equals() : 문자열 비교 **
String str1 = "hello";
String str2 = "hello";

System.out.println(str1.equals(str2));

결과 true

String str1 = "hello";
		String str2 = "helloo";
		System.out.println(str1.equals(str2));
결과 false
```
# private
- private 으로 선언된 필드는 외부에서 접근이 안된다

# setter&getter
- private으로 선언된 필드에 접근할 수 있도록 해주는 메서드.
- 조건 검사  setter에서 한다

# void
- 리턴값(돌려주는 값)이 없다
- 결과 없이 실행만 하는 메서드

# static
- static = 객체 없이 클래스 이름으로 바로 쓰는 것

# extends
- 상속을 받는쪽이 상속을 하는쪽을 선택한다.

# final
- 한 번 정하면 못 바꾼다 (변경 불가) 
```java
final int num = 10;
num = 20; // ❌ 오류

 값 변경 불가
```
# static + final 같이 쓰는 이유
- 모든 곳에서 쓰고, 절대 안 바뀌게 하려고
